# v0.1c — Windows Tray App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a polished Windows system-tray app for the mail-mcp daemon, feature-equivalent to the v0.1b macOS tray.

**Architecture:** New `tray-app-win/` Visual Studio solution. **Phase A is pure WinForms .NET 8** (the original WinUI 3 + WinForms hybrid hit MC6000 in CI under three different csproj-level fixes — see "Phase A pivot" note below). The daemon binary is bundled inside the install dir (`<install>\mail-mcp-daemon.exe`). Tray holds zero persistent state — every fact comes from the daemon over named-pipe JSON-RPC (the v0.1c-blocking IPC port shipped in PR #11). Wizard / Settings windows (Phase B onward) run in a separate WinUI 3 sub-project so the WPF/WinForms target collision stays isolated.

**Tech Stack:** .NET 8, `System.Windows.Forms.NotifyIcon`, `System.IO.Pipes.NamedPipeClientStream`, `CommunityToolkit.Mvvm` (Phase A). WinUI 3 + Windows App SDK 1.5+ join in Phase B as a sub-project for the wizard / settings windows. WiX 4 for the MSI installer, signtool for code signing. CI runs on `windows-latest` runners (native MSVC).

> **Phase A pivot (post-PR-#13):** The original architecture called for `<UseWinUI>true</UseWinUI>` + `<UseWindowsForms>true</UseWindowsForms>` in a single project. `UseWindowsForms=true` transitively imports the WindowsDesktop SDK + Microsoft.WinFX.targets, whose WPF MarkupCompilePass1 fires on any `App.xaml` and emits MC6000 because we don't reference PresentationCore. `<UseWPF>false</UseWPF>`, inline `<Target Name="MarkupCompilePass1" />` stubs, and the same stubs in `Directory.Build.targets` all failed to suppress it. Phase A pivots to pure WinForms (no `App.xaml`, no `Microsoft.WindowsAppSDK` reference, no `<UseWinUI>`). Phase B+ tasks below that build wizard / settings windows must add a SECOND .NET project (e.g. `MailMCP.WinUI/`) targeting `net8.0-windows10.0.19041.0` with `<UseWinUI>true</UseWinUI>` + Microsoft.WindowsAppSDK, and reference it from `MailMCP.csproj`. The csproj snippets in Tasks 1 / 4-7 below describe the original (WinUI 3) intent — see PR #13 commit history for the WinForms shape that actually shipped.

### Task 2: cargo→exe build script

**Files:**
- Create: `tray-app-win/BuildScripts/build-daemon.ps1`

- [ ] **Step 1: build-daemon.ps1**

```powershell
# Builds mail-mcp-daemon.exe and copies it next to the .NET output.
# Invoked from MailMCP.csproj's BeforeBuild target.

param(
  [string]$Configuration = "Debug",
  [string]$Platform = "x64",
  [string]$OutDir = ""
)

$ErrorActionPreference = "Stop"
$WorkspaceRoot = Join-Path $PSScriptRoot ".." | Join-Path -ChildPath ".." | Resolve-Path
$Daemon = "mail-mcp-daemon"

# Resolve Rust target triple from the .NET platform.
switch ($Platform) {
  "x64"   { $Target = "x86_64-pc-windows-msvc" }
  "arm64" { $Target = "aarch64-pc-windows-msvc" }
  default { throw "Unsupported platform: $Platform" }
}

Push-Location $WorkspaceRoot
try {
  if ($Configuration -eq "Release") {
    cargo build --release --target $Target -p $Daemon
    $BuiltExe = Join-Path $WorkspaceRoot "target\$Target\release\$Daemon.exe"
  } else {
    cargo build --target $Target -p $Daemon
    $BuiltExe = Join-Path $WorkspaceRoot "target\$Target\debug\$Daemon.exe"
  }
} finally {
  Pop-Location
}

if (-not (Test-Path $BuiltExe)) {
  throw "Daemon binary not produced at $BuiltExe"
}

if (-not $OutDir) { throw "OutDir parameter required" }
$Dest = Join-Path $OutDir "$Daemon.exe"
Copy-Item $BuiltExe $Dest -Force
Write-Host "Bundled $Daemon → $Dest"
```

```csharp
using System.Text.Json.Serialization;

namespace MailMCP.IPC;

public sealed record AccountID(string Value)
{
    public override string ToString() => Value;
}

public enum AccountStatus
{
    [JsonStringEnumMemberName("ok")] Ok,
    [JsonStringEnumMemberName("needs_reauth")] NeedsReauth,
    [JsonStringEnumMemberName("network_error")] NetworkError,
}

public sealed record AccountListItem(
    [property: JsonPropertyName("id")] string Id,
    [property: JsonPropertyName("label")] string Label,
    [property: JsonPropertyName("provider")] string Provider,
    [property: JsonPropertyName("email")] string Email,
    [property: JsonPropertyName("status")] AccountStatus Status);

public sealed record AccountAddOAuthInProgress(
    [property: JsonPropertyName("challenge_id")] string ChallengeId,
    [property: JsonPropertyName("auth_url")] string AuthUrl);

public enum Policy
{
    [JsonStringEnumMemberName("allow")] Allow,
    [JsonStringEnumMemberName("confirm")] Confirm,
    [JsonStringEnumMemberName("session")] Session,
    [JsonStringEnumMemberName("draftify")] Draftify,
    [JsonStringEnumMemberName("block")] Block,
}

public enum Category
{
    [JsonStringEnumMemberName("read")] Read,
    [JsonStringEnumMemberName("modify")] Modify,
    [JsonStringEnumMemberName("trash")] Trash,
    [JsonStringEnumMemberName("draft")] Draft,
    [JsonStringEnumMemberName("send")] Send,
}

public sealed record PermissionMap(
    [property: JsonPropertyName("read")] Policy Read,
    [property: JsonPropertyName("modify")] Policy Modify,
    [property: JsonPropertyName("trash")] Policy Trash,
    [property: JsonPropertyName("draft")] Policy Draft,
    [property: JsonPropertyName("send")] Policy Send);

public sealed record McpEndpointInfo(
    [property: JsonPropertyName("url")] string Url,
    [property: JsonPropertyName("bearer_token")] string BearerToken,
    [property: JsonPropertyName("stdio_shim_path")] string? StdioShimPath);

public sealed record DaemonStatus(
    [property: JsonPropertyName("version")] string Version,
    [property: JsonPropertyName("uptime_secs")] ulong UptimeSecs,
    [property: JsonPropertyName("account_count")] uint AccountCount,
    [property: JsonPropertyName("mcp_paused")] bool McpPaused,
    [property: JsonPropertyName("onboarding_complete")] bool OnboardingComplete);

public sealed record PendingApproval(
    [property: JsonPropertyName("id")] string Id,
    [property: JsonPropertyName("account")] string Account,
    [property: JsonPropertyName("category")] Category Category,
    [property: JsonPropertyName("summary")] string Summary,
    [property: JsonPropertyName("details")] System.Text.Json.JsonElement Details,
    [property: JsonPropertyName("created_at")] string CreatedAt,
    [property: JsonPropertyName("expires_at")] string ExpiresAt);

public abstract record DaemonNotification
{
    public sealed record ApprovalRequested(PendingApproval Approval) : DaemonNotification;
    public sealed record ApprovalResolved(string Id, string Decision) : DaemonNotification;
    public sealed record AccountAdded(System.Text.Json.JsonElement Account) : DaemonNotification;
    public sealed record AccountRemoved(string AccountId) : DaemonNotification;
    public sealed record AccountNeedsReauth(string AccountId) : DaemonNotification;
    public sealed record McpPausedChanged(bool Paused) : DaemonNotification;
    public sealed record Unknown(string Method, System.Text.Json.JsonElement Params) : DaemonNotification;

    public static DaemonNotification Decode(System.Text.Json.JsonElement frame)
    {
        var method = frame.GetProperty("method").GetString() ?? "";
        var p = frame.GetProperty("params");
        return method switch
        {
            "approval.requested" => new ApprovalRequested(
                System.Text.Json.JsonSerializer.Deserialize<PendingApproval>(p)!),
            "approval.resolved" => new ApprovalResolved(
                p.GetProperty("id").GetString()!, p.GetProperty("decision").GetString()!),
            "account.added" => new AccountAdded(p.Clone()),
            "account.removed" => new AccountRemoved(p.GetProperty("account_id").GetString()!),
            "account.needs_reauth" => new AccountNeedsReauth(p.GetProperty("account_id").GetString()!),
            "mcp.paused_changed" => new McpPausedChanged(p.GetProperty("paused").GetBoolean()),
            _ => new Unknown(method, p.Clone()),
        };
    }
}
```

```csharp
using System.Text.Json;
using MailMCP.IPC;
using Xunit;

namespace MailMCP.Tests;

public class ModelsTests
{
    [Fact]
    public void AccountListItemDecodesSnakeCase()
    {
        const string json = @"[{""id"":""01H123"",""label"":""Personal"",""provider"":""gmail"",""email"":""alice@example.com"",""status"":""ok""}]";
        var items = JsonSerializer.Deserialize<AccountListItem[]>(json)!;
        Assert.Single(items);
        Assert.Equal("alice@example.com", items[0].Email);
        Assert.Equal(AccountStatus.Ok, items[0].Status);
    }

    [Fact]
    public void NotificationApprovalRequestedDecodes()
    {
        const string json = @"{
            ""method"":""approval.requested"",
            ""params"":{
                ""id"":""01H4567"",""account"":""01H123"",""category"":""send"",
                ""summary"":""send_message"",""details"":{""to"":[""a@b.com""]},
                ""created_at"":""2026-05-01T00:00:00Z"",""expires_at"":""2026-05-01T00:05:00Z""
            }
        }";
        var doc = JsonDocument.Parse(json);
        var n = DaemonNotification.Decode(doc.RootElement);
        var ar = Assert.IsType<DaemonNotification.ApprovalRequested>(n);
        Assert.Equal("01H4567", ar.Approval.Id);
        Assert.Equal(Category.Send, ar.Approval.Category);
    }

    [Fact]
    public void NotificationMcpPausedChangedDecodes()
    {
        const string json = @"{""method"":""mcp.paused_changed"",""params"":{""paused"":true}}";
        var doc = JsonDocument.Parse(json);
        var n = DaemonNotification.Decode(doc.RootElement);
        var pc = Assert.IsType<DaemonNotification.McpPausedChanged>(n);
        Assert.True(pc.Paused);
    }
}
```

```csharp
namespace MailMCP.IPC;

/// Mirrors the Windows branch of mail-mcp-core::paths::Paths::default_for_user.
/// Override via MAIL_MCP_ROOT env var.
public sealed record MailMCPPaths(string DataDir, string LogsDir, string CacheDir, string RuntimeDir, string IpcPipe)
{
    public string EndpointJson => System.IO.Path.Combine(DataDir, "endpoint.json");
    public string PidFile => System.IO.Path.Combine(RuntimeDir, "daemon.pid");
    public string StateDb => System.IO.Path.Combine(DataDir, "state.db");

    public static MailMCPPaths DefaultForUser()
    {
        var root = Environment.GetEnvironmentVariable("MAIL_MCP_ROOT");
        if (!string.IsNullOrEmpty(root))
        {
            return new MailMCPPaths(
                DataDir: System.IO.Path.Combine(root, "data"),
                LogsDir: System.IO.Path.Combine(root, "logs"),
                CacheDir: System.IO.Path.Combine(root, "cache"),
                RuntimeDir: System.IO.Path.Combine(root, "run"),
                IpcPipe: $@"\\.\pipe\mail-mcp-{Environment.UserName}");
        }
        var local = Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData);
        var appDir = System.IO.Path.Combine(local, "mail-mcp");
        return new MailMCPPaths(
            DataDir: appDir,
            LogsDir: System.IO.Path.Combine(appDir, "logs"),
            CacheDir: System.IO.Path.Combine(appDir, "cache"),
            RuntimeDir: System.IO.Path.Combine(appDir, "run"),
            IpcPipe: $@"\\.\pipe\mail-mcp-{Environment.UserName}");
    }
}
```

## Self-review notes

- This plan presupposes PR #11 has merged (named-pipe IPC backend in `mail-mcp-core::ipc::transport`). If it hasn't, the daemon side won't accept Windows pipe connections.
- Sparkle has no Windows equivalent; updates via Velopack or "open Releases page in browser." The MSI also supports re-running to upgrade.
- Code signing on Windows requires either an OV cert (~$200/yr) or an EV cert (~$300/yr + hardware token). Until purchased, ship with a "right-click → Properties → Unblock" instruction in the README.
- Per-user install means no admin elevation, no Windows Service, and the daemon runs under the user's session — matches the macOS architecture.

- Windows Store / MSIX
- Daemon-as-Windows-Service mode
