# v0.4 — Calendar integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add calendar reading + writing to the existing per-account provider infrastructure, with provider parity from day one (Google Calendar for Gmail accounts, Outlook Calendar for M365). IMAP-only accounts have no calendar — that's expected.

**Architecture:** New `Calendar` trait in `mail-mcp-core` parallel to `MailProvider`. Per-provider impls (`gmail::calendar`, `m365::calendar`) sit alongside the existing mail impls. Two new permission categories (`calendar.write`, `calendar.delete`); `calendar.read` reuses the existing `read` category. Tray surface picks up two new permission rows + an "Upcoming events" submenu.

**Tech Stack:** Existing workspace deps. New: `chrono-tz` (IANA timezone DB), `wiremock` for tests. OAuth scope additions wired through the existing `ProviderConfig`.

```rust
// crates/mail-mcp-core/src/calendar/types.rs
use chrono::{DateTime, Utc};
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CalendarSummary {
    pub id: String,
    pub summary: String,
    pub time_zone: String,
    pub is_primary: bool,
    pub access_role: AccessRole,
}

#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum AccessRole { Owner, Writer, Reader, FreeBusyReader }

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CalendarEvent {
    pub id: String,
    pub calendar_id: String,
    pub summary: String,
    pub description: Option<String>,
    pub location: Option<String>,
    pub start: EventTime,
    pub end: EventTime,
    pub attendees: Vec<Attendee>,
    pub status: EventStatus,
    pub conference_data: Option<serde_json::Value>,  // pass-through
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EventTime {
    pub date_time: DateTime<Utc>,
    pub time_zone: String,    // IANA tz name; preserved for round-trip
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Attendee {
    pub email: String,
    pub response_status: ResponseStatus,
    pub optional: bool,
    pub organizer: bool,
}

#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum ResponseStatus { NeedsAction, Accepted, Declined, Tentative }

#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum EventStatus { Confirmed, Tentative, Cancelled }
```

```rust
#[async_trait::async_trait]
pub trait Calendar: Send + Sync {

    async fn delete_event(&self, calendar_id: &str, event_id: &str) -> Result<()>;
    async fn respond(&self, calendar_id: &str, event_id: &str, resp: ResponseStatus) -> Result<()>;
    async fn find_free_slot(&self, attendees: &[String], duration: chrono::Duration, time_min: DateTime<Utc>, time_max: DateTime<Utc>) -> Result<Vec<TimeSlot>>;
}

}
```

```rust
async fn list_calendars_decodes_google_response() {
    let mock = MockServer::start().await;
    Mock::given(method("GET"))
        .and(path("/calendar/v3/users/me/calendarList"))
        .respond_with(ResponseTemplate::new(200).set_body_json(json!({

                { "id": "primary", "summary": "alice@example.com",
                  "timeZone": "America/Los_Angeles", "primary": true,
                  "accessRole": "owner" }
            ]
        })))
        .mount(&mock).await;

    let cal = GmailCalendar::new(/* http with mock.uri() */);
    let cals = cal.list_calendars().await.unwrap();
    assert_eq!(cals.len(), 1);
    assert!(cals[0].is_primary);
    assert_eq!(cals[0].time_zone, "America/Los_Angeles");
}
```
